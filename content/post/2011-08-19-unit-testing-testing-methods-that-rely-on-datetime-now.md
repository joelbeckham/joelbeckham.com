<div class="jive-rendered-content">
  <p>
    I have a class whose behavior depends on the time of day. It called <span style="font-family: &#39;courier new&#39;, courier">DateTime.Now</span> directly to get the current time. Doing it this way makes it difficult to write tests to prove that all use cases work as expected.
  </p>
  
  <p>
    One simple solution is to add a delegate called Now to the class. By default it points to an anonymous method which returns DateTime.Now:

  <p style="padding-bottom: 0px; min-height: 8pt; padding-left: 0px; padding-right: 0px; height: 8pt; padding-top: 0px">
    <p>
      I replaced all references of <span style="font-family: &#39;courier new&#39;, courier">DateTime.Now</span> with <span style="font-family: &#39;courier new&#39;, courier">Now()</span>. The class behaves exactly as it did before.
    </p>
    
    <p>
      The neat thing is that during testing now, I have complete control over the Date and Time. After creation of the class, I can change the behavior of <span style="font-family: &#39;courier new&#39;, courier">Now()</span>. For example, I can set it to a static date:
